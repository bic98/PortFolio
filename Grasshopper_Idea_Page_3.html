03_Extruding a quadrilateral polyline according to given conditions.

Key Ideas:

1. Create multiple curves of different scales and compare their distances from the center of the shape.

2. Perform a true/false check to determine if multiple curves of different scales intersect with each other.

Key Idea 1 : Create multiple curves of different scales and compare their distances from the center of the shape.

You can branch shapes that come within a distance of 3 units from each curve.

Key Idea 2 : Perform a true/false check to determine if multiple curves of different scales intersect with each other.

You can branch geometries that intersect with each curve.

Write the c# script code by using Key Idea 1 and Key Idea 2.

C# Code Implementation:


private void RunScript(List<Curve> x, Plane y, ref object A)
{
    List<List<Curve>> ret = new List<List<Curve>>();
    for (int i = 0; i < 100; i++)
    {
        List<Curve> li = Curve.CreateBooleanUnion(x, 0.01).ToList();
        if (li.Count == 0) break;
        Curve tmp = li[0];
        List<Curve> include = new List<Curve>();
        List<Curve> exclusion = new List<Curve>();
        foreach (var j in x)
        {
            if (Curve.PlanarCurveCollision(tmp, j, y, 0.01))
            {
                include.Add(j);
            }
            else
            {
                exclusion.Add(j);
            }
        }

        ret.Add(include);
        x = exclusion;
    }

    var result = makeDataTree(ret);
    A = result;
}

public static DataTree<T> makeDataTree<T>(List<List<T>> ret)
{
    DataTree<T> tree = new DataTree<T>();
    for (int i = 0; i < ret.Count; i++)
    {
        GH_Path path = new GH_Path(i);
        for (int j = 0; j < ret[i].Count; j++)
        {
            tree.Add(ret[i][j], path);
        }
    }

    return tree;
}

Explain the code

In Grasshopper C# development, I tend to avoid using while loops because they can often lead to infinite loops, which can cause the script to hang and make debugging difficult. Instead, I prefer to use for loops, which can iterate a set number of times, such as 100 to 1000 iterations. This approach is safer and more predictable, especially in a visual scripting environment like Grasshopper.



The side of the code takes an initial set of curves and identifies the largest boundary formed by these curves. It then determines which rectangular cells intersect with this boundary. When an intersection occurs, the code subtracts the intersecting cells from the current set of cells (x) and stores the remaining cells in a different list.



By repeating this process four times, the data is progressively refined, achieving the desired outcome as shown in the referenced diagram.



In some cases, scripting can be more intuitive and concise than using standard Grasshopper components. This is especially true when dealing with complex iterative processes or custom data manipulations, where scripting provides more control and flexibility.
